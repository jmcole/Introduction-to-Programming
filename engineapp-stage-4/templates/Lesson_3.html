Abstraction

Abstraction in programming is a way of managing complexity. The specific way a function works are hidden behind documentation and the details are hidden from the programmer. This allows the programmer to focus on the program we want to write. Wikipedia Article on Abstraction

Take a Break Program

This program demonstrates the use of Python's Standard Library imported modules. 
            
              import time
              import webbrowser
              total_breaks = 3
              break_count = 0
              print("This program started on "+time.ctime())
              while(break count < total_breaks):
                time.sleep(10)
                webbrowser.open("http://www.youtube.com/watch?v=dQw4w9WgXcQ")
                break_count = break_count + 1
How Take a Break Operates
  1. import time and import webbrowser will import the modules from the Python Library
  2. total_break = 3 will be used to break the While loop
  3. break_count = 0 is used to set the value of the counter
  4. time.ctime converts time since the epoch to a string representing the local time
  5. While loop will run 3 times as determined by total_break and break_count
  6. time.sleep suspends the execution of the program for the given number of seconds. Documentation Link
  7. webbrowser.open open the given link in the webrowser
  8. Increments break_count by 1

Make Classes

Creating Classes

Programmers are not limited to just classes that are found externally, or in the Python Standard Library. Programmers can make classes to perform whatever function they need to do.

Creating a Class

  1. Decide on how Class should be used and what parameters it needs.(oversimplified)
  2. Create a new file to write your code for the class. File will be imported into your program like previous examples.
  3. Use Python keyword class
  4. Name the class and capitalize the first letter according to Google Style Guide
  5. Follow with (): className():

def_init_

  1. def_init_(self): is a special function (Contructor)within Python to initilize the newly created empty object.
  2. (self) is the commonly used Python convention for referring to the instance's attribute.
  3. Use self instead of another variable so that it is not mistaken for a local variable.
  4. self is an instance variable, not a local variable.

Using your Created Class

  1. import class into program

Vocabulary Recap

  1. Class- keyword allows us to make classes(blueprints). Class can have both data and methods.
  2. Instancesare multiple versions of a class.
  3. The Constructor def_init_ initilaizes all of the data associated wih the instance.
  4. self is itself or the instance in question.
  5. instance variables are the variables associated with a specific instance. Unique to the object.
  6. instance methods are the functions within the class that are associated with an instance and have the first argument as self.

Advanced Topics

Class Variables

  2. Class variables are before the def_init_
  3. Constant variables should be "all caps"
  4. All classes come with pre-existing class variables in Python.
  5. _doc_ is a class variable that when used gives "documentaton" on the class
  6. """ triple qoutes used at the beginning of the class will be called by the _doc_

Inheritence

  1. Inheritance allows code to reused through Child Classes "inheriting" traits from Parent Classes.
    2.

                    def_init_(self,last_name,eye_color):
                    self.last_name =last_name
                    self.eye_color = eye_color
                  class Child(Parent:) <---------This creates the inheritance
                    def_init_(self,last_name,eye_color,number_of_toys):

                      self.number_of_toys = number_of_toys
  2. Writing code utilizing inheritance allows us to write intuitivley the way our brain works.

Reusing Methods

  1. If a child inherits variables from the parent, it also inherits methods.
  2. This means a method can be called for the child even though it is defined in the parent class.
  3. This allows a programmer to reuse code.
  4. If the same method is defined in the child it will "method overide" the parent class. In other words, the child method will be used instead of the parent method.
  5. OOP is a way to structure code so that it is intuitive and clean, meaning that code is reused and logical. This allows programs to use less resources, be easier to troubleshoot and modify, and make them easier to design.
